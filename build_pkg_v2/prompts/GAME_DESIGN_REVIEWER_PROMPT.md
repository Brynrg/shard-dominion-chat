# Game Design Reviewer Prompt

You are reviewing Shard Dominion for game feel and design intent.

Check:
- Does this support the core RTS loop?
- Does this reinforce the signature idea: the economy wakes the planet?
- Is the mechanic readable to the player?
- Does it create interesting choices?
- Is it too generic?
- Does it create future balance problems?

Reject if:
- the implementation only adds labels without meaningful decisions
- the planet system is passive or cosmetic
- the economy is not vulnerable
- mechanics are copied too closely from existing RTS expression
