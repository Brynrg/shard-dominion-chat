# UI/UX Specification

## Required screens

1. Boot/loading
2. Main menu
3. Campaign select
4. Mission briefing
5. Skirmish setup
6. Settings
7. In-game HUD
8. Pause menu
9. Victory
10. Defeat
11. Debrief
12. Credits

## In-game layout

Upper-left:
- objectives;
- alerts;
- mission messages.

Upper-right:
- credits;
- storage;
- power;
- global agitation.

Lower-left:
- minimap;
- radar state;
- planet-event alerts.

Lower-center:
- selected unit/building;
- HP;
- cargo;
- veterancy;
- orders;
- weapon/status.

Right sidebar:
- buildings;
- units;
- research;
- abilities.

## Controls

- left click select;
- drag select;
- Shift add/remove;
- double-click same type;
- right click context command;
- A attack-move;
- S stop;
- G guard;
- P patrol;
- R repair where applicable;
- Ctrl+number group;
- number recall group;
- WASD/arrows camera;
- wheel zoom;
- minimap click pan;
- Escape cancel/clear/pause.

## UX requirements

- visible command acknowledgement;
- valid/invalid placement feedback;
- target cursor;
- attack cursor;
- move marker;
- selection rings;
- health bars;
- cargo bar;
- power warnings;
- agitation warning severity;
- tooltips;
- hotkeys in tooltip.

## Accessibility

- scalable UI;
- reduced screen shake;
- colorblind-safe faction markers;
- subtitles;
- volume controls;
- remappable keys if feasible;
- pause;
- readable font sizing.
